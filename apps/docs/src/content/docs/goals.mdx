# Continuer à développer avec Laravel
J’ai démarré un projet projet avec la stack TALL, ce qui m’a permis de reprendre contact avec l’univers PHP.
Avec ce projet, je souhaite continuer dans cette direction, avec Laravel

# Intégrer API Platform
Après les deux jours de conférences de API Plateform à Lille en septembre 2025, j’ai envie d’apprendre à utiliser cet outil, à la base destiné à l’environnement Symfony.
Le but est donc de découpler une application entre son backend et son front

# Le Monorepo
Dans cette ligne, j’aimerais me confronter à l’architecture Monorepo, en utilisant Moonrepo.

# Faire un projet PHP dans le terminal
Plutôt que d’en faire un projet web, j’ai envie d’apprendre à créer un projet qui fonctionne dans le terminal. J’aime beaucoup pouvoir simplement utiliser mon clavier pour naviguer, et je n’ai pas besoin d’un environnement front très compliqué.
Dans un premier temps, j’aimerais en faire un simple script CLI, puis ensuite créer une interface TUI.

Il sera toujours possible après de rajouter une interface web, en React ou en Vue.

# Packager un projet PHP
Ça, je ne sais pas encore exactement comment, mais si je peux découvrir/apprendre comment utiliser FrankenPHP dans cet projet, ça serait topissime

# Documenter et partager
Utiliser Astro pour créer des documents d’avancement, mettre en avant ma Roadmap, revenir sur mes obstacles et mes réussites.

# Et pour faire quoi au fait ?
J’ai besoin d’un outil pour me permettre de prendre des notes. Alors, j’utilise Raindrop.io dans la vraie vie du vrai monde, même si je trouve qu’on ne peut pas vraiment saisir des notes comme on veut.
Et je me suis dit, mais c’est pourtant pas bien compliqué : j’ai une source, j’ai des éléments qui proviennent de cette source que je veux sélectionner, et je veux pouvoir annoter la source. Puis tout retrouver facilement.
C’esc comme ça que je me retrouve à faire une application en ligne de commande pour créer des bibliographies.
